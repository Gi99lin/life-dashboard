## 1. Контекст и проблема

Текущий дашборд (после визуального редизайна, см. §9) выглядит премиально, но **наполнение** слабое:
плоский список одиночных метрик с «детскими» декоративными спарклайнами, без связей,
целей и инсайтов. Карточки не отвечают на вопрос «и что?».

**Задача:** пересобрать смысл и визуализацию наполнения так, чтобы дашборд:
- давал **осмысленную иерархию** (герой → домены → детали), а не «стену цифр»;
- был **прозрачным**: каждый показатель и вывод можно развернуть до первоисточника
  (тултип со значением/средним/источником, клик → графики и корреляции). Никакой «магии»;
- **флексил инженерно**: глубокая dev-активность, real-time, ИИ-инсайты на собственных LLM;
- оставался **дашбордом** (графики/метрики), а не лендингом из 3–5 цифр и вердикта модели.

### Решения, принятые на брейншторме
- **Архитектура:** вариант A (Readiness Hero + Домены), но data-rich и с drill-down. (Не чистый B/bento и не C/console — их элементы заимствуем точечно.)
- **Источники:** углубляем существующие + добавляем **dev-активность** (WakaTime, GitHub-deep, CI) и **ИИ-инсайты** (свои LLM). Музыку (Spotify) и календарь **не берём** — нет единой точки входа.
- **Палитра/айдентика:** наследует портфолио `ivanyakimkin.ru` (зелёный акцент, mono-лейблы, blueprint-сетка) — уже внедрено (§9).

## 3. Экран «Обзор» — состав

Сверху вниз. Каждый элемент имеет источник и drill-down (см. §5).

### 3.1 Шапка (без изменений)
Брендинг + приветствие · панель вкладок · виджет погоды/даты/текущего блока. Одобрено как есть.

### 3.2 Герой — «Состояние дня»
- **Концентрические кольца** (Apple/Oura): композитный балл (0–100) в центре + 4 кольца-фактора
  (Сон, Энергия, Спокойствие, HRV).
- **Контекст:** статус-фраза («Готов к нагрузке»), дельта vs среднее 30д, короткое описание состояния,
  список факторов со значениями, дельтами и **мини-спарклайнами** (14 дн).
- **Полоса готовности за 14 дней** (today подсвечен) + строка-рекомендация (привязка к расписанию).
- **AI-разбор дня** (справа): 1–2 предложения, сгенерированные LLM из чисел дня, с чипами-источниками
  (Garmin / GitHub / WakaTime / Obsidian) и «развернуть →» (показывает, на каких числах построено).

### 3.3 Live-строка (одобрено, без изменений по составу)
4 компактные ячейки: **Сейчас** (текущая активность/фокус, WakaTime) · **По плану** (текущий блок,
Obsidian) · **Инфра** (контейнеры ok, Docker) · **Стрик кода** (GitHub). Обновляется по WebSocket.

### 3.4 Три домена (одобрено)
Каждый домен — карточка с источником, «развернуть →», главной визуализацией и sub-метриками
(range-bars: значение vs персональный диапазон, с ⓘ).

- **Тело** (Garmin): сон в часах + **stage-bar** (Глубокий/Лёгкий/REM/Бодр.); sub: Пульс покоя, SpO₂, Body Battery.
- **Разум** (Obsidian · Garmin): настроение 1–5 + спарклайн; sub: Стресс ср, Фокус, связь (r) сон↔настроение.
- **Работа** (WakaTime · GitHub): часы кода + **донат по языкам**; sub: Коммиты (streak), PR смержено/review, Deep-work.

### 3.5 Низ — два содержательных блока
- **Корреляции:** компактная **матрица** (heatmap, зелёный +/красный −, насыщенность = сила; приглушённая,
  supporting-роль) + список **«Сильнейшие связи»** с человекочитаемой расшифровкой. Клик по ячейке → скаттер в Аналитике.
- **Тренды · 30д:** настоящий мульти-серийный график (градиентная заливка, сетка, маркер «сегодня»,
  легенда, ось дат). Наведение → значение дня, клик → развернуть.

## 4. Композитный балл «Состояние дня» (прозрачная формула)

Балл **не магия**: считается в коде, веса видимы, факторы показаны рядом.

```
Readiness = round( wS·Sleep + wE·Energy + wC·Calm + wH·HRV )
```

Компоненты нормируются в 0–100:
| Фактор | Источник | Нормализация (v1, калибруется) |
|---|---|---|
| Sleep | Garmin `sleep_score` (или дериватив часы+стадии) | как есть 0–100 |
| Energy | Garmin `body_battery_max` | как есть 0–100 |
| Calm | `100 − stress_avg` | стресс 0–100, инвертируем |
| HRV | Garmin HRV vs персональный baseline | перцентиль/скейл к 0–100 |

**Веса v1 (иллюстративные, тюним при разработке):** `wS=0.35, wE=0.30, wC=0.20, wH=0.15`.
Клик по герою → разворачивает формулу, вклад каждого фактора, тренды компонентов.
Если данных по фактору нет — он исключается, веса перенормируются (балл честно отражает доступное).

> Открытый вопрос: точные веса и нормализация HRV — подберём на реальных данных в разработке.

## 5. Прозрачность и drill-down (ключевое требование)

Единая модель «всё разворачивается до первоисточника»:

- **Тултип** (наведение на любую метрику/ячейку): текущее значение, среднее за период, диапазон, **источник**.
- **Клик** по метрике → соответствующий раздел **Аналитики**: полный график истории + корреляции этой метрики.
- **Чипы-источники** (Garmin/GitHub/WakaTime/Obsidian/Docker) — кликабельны, ведут на первоисточник
  (приложение/профиль/файл Obsidian), где это применимо.
- **AI-разбор** «развернуть →» показывает числа, на которых построен вывод (трассируемость).
- **Матрица корреляций** «открыть в Аналитике →» — большой вид с выбором пар и скаттерами.

Принцип: на Обзоре — сжатый сигнал; за один клик — доказательная база.

## 6. Источники данных и коллекторы

### 6.1 Существующие (углубляем визуализацию)
Garmin (здоровье), Git (коммиты), Open-Meteo (погода), Obsidian (расписание/настроение/заметки), Docker/инфра.

### 6.2 Новые коллекторы
- **WakaTime** (`collector/sources/wakatime_source.py`): дневные саммари → часы кода всего, по языкам,
  по проектам, фокус-часы. Хранение: `day.wakatime = { total_h, by_language{}, by_project{}, focus_h }`.
- **GitHub-deep** (расширить `git_source.py` или новый `github_source.py`): PR открыто/смержено, ревью,
  языки, стрик, lines +/−. Хранение: `day.github = { commits, prs_merged, reviews, streak, langs{}, additions, deletions }`.
- **CI/CD** (GitHub Actions API или своя инфра): статусы сборок/деплоев. На Обзоре — в live-строке/Инфре.
- **AI-инсайты** (свои LLM через OpenAI-совместимый эндпоинт / LibreChat backend):
  генератор берёт метрики за период, корреляции считаются **в коде** (Pearson по последним N дням),
  LLM **интерпретирует** топ-связи и пишет дневной бриф. Хранение: `day.ai = { brief, top_correlations[], generated_at, sources[] }`.

### 6.3 Прайваси
Все источники self-hosted / личные ключи; ИИ-инсайты идут через **собственные** LLM (данные не уходят наружу).

К текущему `day { date, week_iso, manual, garmin, git, schedule }` добавляются:
```jsonc

  "wakatime": { "total_h": 4.8, "by_language": { "TypeScript": 1.9, "Python": 1.3 }, "focus_h": 4.1 },

  "readiness":{ "score": 76, "sleep": 82, "energy": 71, "calm": 66, "hrv": 58 },  // кэш расчёта

}
```
`readiness` и `ai` могут считаться на лету в API, либо кэшироваться в `metrics.json` коллектором.
